Philosophy

Embrace the legacy

Legacy systems earned that name by surviving. They run payroll, track inventory, process orders. They work. The instinct to replace them is expensive, risky, and usually wrong.

The real problem is never the system itself. It's the gap between what it holds and what the rest of the organisation needs from it. We close that gap.

Understand what works

Before writing any code, we map what the existing systems do well and where the friction lives. The goal is integration, not replacement.

Choose the right tool

Sometimes that's an anti-corruption layer with CDC and event streaming. Sometimes it's a simple adapter. Sometimes it's a conversation about what you actually need. We're not here to sell a platform.

Leave you with something that lasts

Standard tools, standard infrastructure, complete source ownership. Nothing that requires us to stay. The best engagement is one your team can maintain alone.

Perspective

Speed is a distraction

Teams spend months debating Rust vs Java vs TypeScript based on benchmarks that vanish behind a single database query. We built the same 13-entity federated API in all three languages, against the same MongoDB infrastructure. Under load, they converge.

The difference between "0.7ms" and "1.8ms" is undetectable to any human or frontend. The real performance lever is infrastructure—indexing, connection pooling, sharding—not language choice. The right question isn't "which language is fastest?" It's "which language lets this team ship reliable software fastest?"

Open Source

Tools we build and use

We build tools when the right one doesn't exist. They're available under BSL 1.1—free for non-production use, commercial license for production.

MeshQL

Java 21 · GraphQL + REST

Data API framework for building federated services from schema definitions. Supports PostgreSQL, MongoDB, and SQLite. Temporal queries, multi-backend federation, full CRUD.

Documentation Source

Mesher

CLI · Code generation

Points at a legacy database, introspects the schema, and generates a complete anti-corruption layer project. The fastest way to get a working integration from an existing system.

Documentation Source

merkql

Rust · Embedded event log

Kafka semantics backed by a merkle tree. Topics, partitions, consumer groups, and cryptographic inclusion proofs—without the cluster. For audit logs, event sourcing, edge systems, and testing.

Documentation Source

meshobj

TypeScript

Type-safe client for MeshQL APIs. Zero dependencies.
